```

- pytest --cov=magicsquare --cov-report=term-missing 실행
- 커버되지 않은 줄(Missing) 중 Invariant 관련 분기 우선 식별
- Domain Logic 95% 미달 시: 누락된 테스트 케이스 제안 (구현 코드 추가 금지)
- Boundary 85% 미달 시: 누락된 계약 테스트 케이스 제안

  | 파일 | Stmts | Miss | Cover | 미달 원인 | 추가 테스트 제안 |
```

### Assistant 산출 요약

| 항목 | 결과 |
|------|------|
| **`--cov=magicsquare`** | ❌ `Module magicsquare was never imported` — 커버리지 0건 |
| **`domain.py` / `boundary.py`** | ❌ 미존재 → ECB 실경로 매핑 필요 |
| **pytest 기준선** | 32 PASS / 19 RED (`pytest.fail` 스켈레톤) |
| **Domain (entity+control)** | 91% — Gate 95% **FAIL** |
| **Boundary (전체)** | 41% — Screen 0% 포함 — Gate 85% **FAIL** |
| **Boundary (계약만)** | 96% — screen 제외 시 PASS |
| **핵심 갭** | `MagicSquareValidator` 81%, RED 19건 ↔ Missing Invariant 분기 |

**프롬프트 수정 포인트:**

- `--cov=src/entity --cov=src/control` / `--cov=src/boundary` / `--cov=src`
- ECB 경로 매핑 표 + Step 0 (pytest + GM-1) 추가
- RED 스켈레톤 GREEN vs 신규 테스트 구분
- 출력 A~E 형식 (기준선·요약·파일별·Invariant·판정)
