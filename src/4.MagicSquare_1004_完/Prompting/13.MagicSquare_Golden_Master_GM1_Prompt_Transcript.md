# Turn 1 — GM-1 기준 파일 생성 (GM-01~GM-06)

### User Prompt

```
현재 Magic Square Solver의 실제 출력을 기반으로 Golden Master 기준 파일을 생성하고 버전 관리에 포함한다.
🔲 GM-1 기준 파일 생성 — Magic Square
[P] 회귀 테스트(Approval/Golden Master) 설계 전문가입니다.
[C] pytest/JUnit/Catch2 기반 테스트 환경
[T] Magic Square Solver 출력 기반 Golden Master 기준 파일을 생성해줘.

구현 요구사항:
1. 기준 출력 파일 생성 전략
   입력 시나리오:
   • 정상 조합 성공
   • reverse 조합 성공
   • invalid blank count
   • duplicate number
   • no valid solution
   출력 캡처: stdout capture 또는 Result DTO serialize
   생성 후: git add tests/golden_master_expected.txt

2. approve 패턴 적용
   • 기준 파일 없으면: 현재 출력을 기준으로 자동 생성
   • 기준 파일 있으면: actual vs expected 비교
   • 불일치 시: unified diff 출력 후 FAIL

3. 기준 파일 구조
   [normal_success] Input: ... Output: [3,3,1,4,4,6]
   [reverse_success] ...
   [invalid_blank_count] Error: INVALID_BLANK_COUNT

[F] tests/golden_master_expected.txt + 생성 스크립트 + approve 패턴 설계 문서
```

### Assistant 산출 요약

| 항목 | 결과 |
|------|------|

| **Approve** | `approve_golden_master()` — auto-create / unified diff / `--approve-golden` |
| **스크립트** | `scripts/generate_golden_master.py` (+ `--check`) |
| **설계 문서** | `docs/golden_master_design.md` |
| **pytest** | `tests/test_gm_01_magic_square_golden_master.py` — **1 PASS** |
| **git** | `golden_master_expected.txt` 및 관련 파일 **staged** |
