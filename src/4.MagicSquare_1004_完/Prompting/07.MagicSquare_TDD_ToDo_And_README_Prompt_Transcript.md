# Phase A — TDD 개발 To-Do List

### User Prompt

```
당신은 Senior Software Architect이자 Dual-Track UI + Logic TDD 코치입니다.

목표:
MagicSquare 4x4 프로젝트의 구현을 시작하기 전에,

이 To-Do List는 단순 작업 목록이 아니라,
다음 TDD 흐름을 추적하는 개발 보드입니다.

Scenario
→ Acceptance Criteria
→ RED Test ID
→ Test Skeleton
→ Run Test
→ Confirm Failure = RED
→ Minimal Implementation = GREEN
→ Structure Improvement = REFACTOR

중요:
- 지금 구현 코드를 작성하지 마십시오.
- 지금 테스트 코드를 작성하지 마십시오.
- 지금 파일을 생성하거나 수정하지 마십시오.
- To-Do List 본문만 Markdown으로 출력하십시오.
- RED는 반드시 "실패 상태 확인"으로 표현하십시오.
- GREEN Task는 실제 구현이 아니라 "RED 실패를 통과시키기 위한 최소 구현 작업 후보"로만 작성하십시오.
- REFACTOR는 실제 수행하지 말고 "GREEN 이후 구조 개선 후보"로만 작성하십시오.

(참고 문서: docs/PRD, Report/01~06, .cursorrules)
(필수 Scenario 15건, 출력 형식: Epic, US, Tracking Board 표, RED Checklist, Traceability Summary)
```
