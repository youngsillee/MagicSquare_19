# STEP 1 — Agent 프롬프트 설계 (파일 생성 금지)

### User Prompt

```
아래 역할별 Cursor Agent 프롬프트를 작성해 주세요.

목표:
MagicSquare 4x4 TDD 연습 프로젝트에서 사용할 Agent 프롬프트 세트를 만든다.

저장 위치 규칙:
- 모든 Agent 프롬프트는 프로젝트 루트의 .cursor/agents/ 폴더에 저장한다.
- Agent별로 하나의 Markdown 파일을 사용한다.
- 파일명은 Agent 이름과 동일하게 한다.
- 각 Agent 출력 앞에 저장 경로를 명시한다.
- 단, 지금은 파일을 실제로 생성하지 말고, 복사 가능한 텍스트만 출력한다.

저장 경로:
1. .cursor/agents/system-optimization-engineer.md
2. .cursor/agents/ux-design-advisor.md
3. .cursor/agents/product-planning-manager.md
4. .cursor/agents/backend-developer.md
5. .cursor/agents/frontend-developer.md
6. .cursor/agents/quality-assurance-engineer.md
7. .cursor/agents/ai-integration-expert.md
8. .cursor/agents/backup-report-github-manager.md

프로젝트:
- 프로젝트명: MagicSquare
- 언어: Python 3.10+
- 테스트 프레임워크: pytest
- 테스트 패턴: AAA, Arrange-Act-Assert
- 아키텍처: ECB, Entity-Control-Boundary
- TDD 방식: Dual-Track UI + Logic TDD
- 주요 목적: 계층 분리, 계약 기반 테스트, RED-GREEN-REFACTOR, 안전한 리팩토링 연습

MagicSquare 도메인 규칙:
- 입력은 4x4 int 행렬이다.
- 0은 빈칸을 의미한다.
- 빈칸은 정확히 2개여야 한다.
- 값의 범위는 0 또는 1부터 16까지이다.
- 0을 제외한 숫자는 중복될 수 없다.
- 출력은 int[6]이다.
- 좌표는 1-index 기준이다.
- 출력 형식은 [r1, c1, n1, r2, c2, n2]이다.
- 4x4 마방진 상수는 34이다.
- 먼저 작은 누락 숫자를 첫 번째 빈칸에, 큰 누락 숫자를 두 번째 빈칸에 넣어 시도한다.
- 해당 조합이 마방진을 만들지 못하면 반대 조합을 시도한다.

생성할 Agent:
1. system-optimization-engineer
2. ux-design-advisor
3. product-planning-manager
4. backend-developer
5. frontend-developer
6. quality-assurance-engineer
7. ai-integration-expert
8. backup-report-github-manager

각 Agent는 아래 형식으로 작성한다.

# 저장 경로
# Agent Name
# Role
# Responsibilities
# Workflow
# Must Not
# Output Format

공통 안전 규칙:
- 사용자 승인 없이 파일 삭제, 대량 파일 이동, Git push, 배포, DB 또는 저장소 변경을 하지 않는다.
- API Key, token, password, secret을 출력하거나 커밋하지 않는다.
- 코드를 수정하기 전에 관련 파일을 먼저 확인한다.
- 프로덕션 코드를 작성하기 전에 관련 테스트를 먼저 확인한다.
- 수정 후에는 변경된 파일 목록과 테스트 결과를 보고한다.
- 확인할 수 없는 부분은 "확인 필요"라고 표시한다.
- 근거 없이 추측으로 코드를 수정하지 않는다.
- 테스트를 통과시키기 위해 테스트를 약화, 삭제, 우회하지 않는다.
- ECB 계층 경계를 위반하지 않는다.
- 타입 힌트 없는 함수를 생성하지 않는다.
- print() 디버깅을 사용하지 않는다.
- PEP8을 준수한다.
- 공개 메서드에는 Google 스타일 docstring을 작성한다.
- RED-GREEN-REFACTOR 순서를 엄격히 따른다.
- GREEN 단계에서는 대상 실패 테스트를 통과시키기 위한 최소 코드만 작성한다.
- REFACTOR 단계에서는 외부 동작이나 계약을 변경하지 않는다.

중요:
- 지금 실제 MagicSquare 애플리케이션 코드를 수정하지 않는다.
- 지금 파일을 생성하지 않는다.
- 지금 구현 코드를 작성하지 않는다.
- 지금 실행 가능한 테스트 코드를 작성하지 않는다.
- 오직 Agent 프롬프트만 작성한다.
- 복사해서 붙여넣을 수 있는 텍스트만 출력한다.

추가 TDD 제약:
- UI/Boundary RED 테스트와 Logic/Domain RED 테스트를 분리한다.
- Boundary 계층 테스트는 입력, 출력, 오류 계약을 보호한다.
- Domain 계층 테스트는 순수 마방진 불변식을 보호한다.
- Boundary는 Control을 호출할 수 있다.
- Control은 Entity를 호출할 수 있다.
- Entity는 Control이나 Boundary에 의존하면 안 된다.
- Domain 로직은 Boundary에 구현하면 안 된다.
- 입력 검증 계약과 마방진 해 결정 로직을 섞지 않는다.
```
