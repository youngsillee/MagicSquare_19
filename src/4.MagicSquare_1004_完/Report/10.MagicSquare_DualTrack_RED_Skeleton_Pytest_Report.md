본 세션은 Report/09 Dual-Track RED 설계표(24건) 중 **pytest가 없던 항목**에 대해 **RED Skeleton** 테스트를 작성하고, `pytest` 실행으로 실패 상태를 확인했다. production 코드(`src/boundary/`, `src/entity/services/`)는 작성하지 않았다.

| Phase | 내용 | 산출 |

## 8. pytest·커버리지 명령 가이드

### 테스트 실행

```bash
# 전체
python -m pytest

# Track별
python -m pytest tests/boundary/ -v
python -m pytest tests/entity/ -v
python -m pytest tests/boundary/ tests/entity/ -v
```

### 커버리지 (pytest-cov)

```bash
pip install pytest-cov

# 전역 (REFACTOR gate ≥ 80%)
python -m pytest --cov=src --cov-report=term-missing

# Boundary ≥ 85%
python -m pytest tests/boundary/ --cov=src/boundary --cov-report=term-missing --cov-fail-under=85

# Domain ≥ 95%
python -m pytest tests/entity/ tests/control/ --cov=src/entity --cov=src/control \
  --cov-report=term-missing --cov-fail-under=95

# HTML 리포트
python -m pytest tests/entity/ --cov=src --cov-report=html
# → htmlcov/index.html
```

| 상황 | 참고 |
|------|------|
| Boundary collection ERROR (현재) | `--cov` HTML 미생성 가능 — Entity만 먼저 측정 |
| 측정 대상 | `src/` only; `tests/` 제외 |
