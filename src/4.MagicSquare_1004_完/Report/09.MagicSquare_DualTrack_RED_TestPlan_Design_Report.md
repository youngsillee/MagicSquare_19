## 1. 개요

본 세션은 Magic Square 4×4 프로젝트 **Dual-Track TDD RED phase**에서 FR-01~FR-05 범위의 **실패 전제 테스트 설계표**를 작성했다. 구현 코드·pytest 테스트·스켈레톤 코드는 의도적으로 포함하지 않았다.

| Phase | 내용 | 산출 |
|-------|------|------|
| **A** | 초안 RED 프롬프트 검증 및 SSOT 정합 개선안 제시 | 검증 결과 + 개선 프롬프트 (채팅) |
| **B** | FR-01~FR-05 Dual-Track RED 설계표 작성 | UI RED 12건 + Logic RED 12건 (본 Report) |
| **C** | Report/Transcript Export | Report/09, Prompting/09 |

### 세션 TDD phase

| 항목 | 값 |
|------|-----|
| **현재 phase** | **RED** (테스트 **설계**만 — pytest 미작성) |
| **Track A** | Boundary — U-IN-01~08, U-OUT-01~03, U-FLOW-02 |
| **Track B** | Domain — D-LOC-01, D-MIS-01, D-VAL-01~06, D-SOL-01~04 |
| **GREEN** | 미진입 |
| **REFACTOR** | 미진입 |

### SSOT 계약 (본 설계표 기준)

| 항목 | 규칙 |
|------|------|
| 입력 | 4×4 `int[][]`, `0`=빈칸(정확히 2개), `0` 또는 `1~16`, non-zero 중복 금지 |
| 출력 | `int[6]` = `[r1,c1,n1,r2,c2,n2]`, 좌표 1-index |
| 마방진 상수 | M=34 (`MagicConstant` SSOT) |
| 검증 순서 | null → size → empty count → value range → duplicate |
| Boundary invalid | Failure envelope (E001~E005); Python 예외 아님 |
| Domain 격리 | invalid → `SolvePartialMagicSquare.execute` **0회** |
| Logic Track | Domain Mock **금지** |
