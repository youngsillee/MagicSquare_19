## 1. 개요

본 세션은 Magic Square 4×4 프로젝트 **구현 착수 전**에 사용할 PRD를 작성하기 위해 3단계를 수행했다.

| Phase | 내용 | 산출 |
|-------|------|------|
| **A** | Report/01~05, `.cursorrules` 분석 → PRD 섹션 매핑 | Reference Document Analysis |
| **B** | 23섹션 구조 PRD 본문 작성 | PRD v0.1 Draft (채팅) |
| **C** | 7기준 PRD 품질 검토 | Review Report (문제·개선안) |

PRD는 “기능 설명서”가 아니라 **불변식 기반 사고 훈련, I/O 계약 고정, Boundary/Domain 분리, Dual-Track TDD, Concept→Code Traceability**를 보장하는 구현 전 기준 문서로 정의되었다.

## 6. PRD 핵심 산출 요약

### 6.1 Problem Statement (한 문장)

> 빈칸 2개 부분 격자를 입력받아 누락 숫자 2개의 유효 배치를 결정하고 `int[6]`(1-index)으로 반환한다. 입력 위반은 Boundary(E001~E005), 해결 불가는 E006.

### 6.2 Scope

**In-Scope:** US-01~05, Dual-Track U-*/D-*, IT-01/03/E*  
**Out-of-Scope:** UI 화면, DB, Web/API, N×N, 완전 생성, User Entity, Data Layer (Phase 2)

### 6.3 Architecture (ECB)

```
Caller → Boundary (Track A) → Control → Domain (Track B) → Boundary → int[6]
```

금지: entity → boundary/control, control → boundary
