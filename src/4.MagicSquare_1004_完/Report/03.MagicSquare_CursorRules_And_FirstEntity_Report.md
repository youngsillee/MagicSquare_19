## 1. Cursor Rule 설계 결정

### 1.1 `.cursorrules` vs `.cursor/rules/*.mdc`

| 방식 | 장점 | 단점 |
|------|------|------|
| **`.cursorrules` (단일 YAML)** | 초기 설정 빠름, 한 파일에서 전체 규칙 조망 | glob별 선택 적용 불가, 규칙 증가 시 토큰·우선순위 혼동 |
| **`.cursor/rules/*.mdc` (권장)** | `alwaysApply` / `globs` 분리, Rule Picker 지원 | 파일 수 증가 |

**본 프로젝트 선택:** 우선 **`.cursorrules` 단일 YAML** 로 프로젝트 규칙 SSOT를 확립.  
향후 규칙이 커지면 `.mdc` 분할(`00-core`, `pytest-aaa`, `ecb-architecture` 등)을 검토.

### 1.2 Rule 파일 분할안 (향후 `.mdc` 전환 시)

| 파일 | alwaysApply | globs | 내용 |
|------|-------------|-------|------|
| `00-project-core.mdc` | true | — | RED 우선, Dual-Track, 테스트 약화 금지 |
| `01-python-standards.mdc` | false | `**/*.py` | PEP8, type hints, Black 88 |
| `02-ecb-architecture.mdc` | false | `src/**/*.py` | ECB 의존 방향 |
| `03-dual-track-tdd.mdc` | true | — | Logic/UI Track Mock 규칙 |
| `04-pytest-aaa.mdc` | false | `tests/**/*.py` | AAA, fixture scope |

## 2. `.cursorrules` 파일 구조

루트에 `.cursorrules` 1개, **8개 최상위 키**, 각 키 앞 **80자 `#` 구분선** 주석.

```
project → code_style → architecture → tdd_rules → testing
       → forbidden → file_structure → ai_behavior
```

| 섹션 | 역할 |
|------|------|
| `project` | 프로젝트명, 입출력 계약, Dual-Track, SSOT Report 경로 |
| `code_style` | Python 3.10+, PEP8, type hints, Google docstring, Black 88 |
| `architecture` | ECB 3레이어, 의존 방향, Dual-Track |
| `tdd_rules` | red / green / refactor phase (description, rules, must_not) |
| `testing` | pytest, AAA, coverage 80%, fixture scope, naming |
| `forbidden` | pattern / reason / alternative 구조 |
| `file_structure` | ECB 기준 디렉터리 트리 |
| `ai_behavior` | 코드 작성 전·중·후 에이전트 행동 규칙 |

## 3. `.cursorrules` 검토 결과

검토 시점: `tdd_rules`만 작성된 중간 상태.

| 항목 | 결과 |
|------|------|
| YAML 문법 | 치명적 오류 없음 (빈 키 7개는 null 상태) |
| 누락 섹션 | 7개 섹션 내용 비어 있음 → 이후 전 섹션 완성 |
| `tdd_rules` ↔ `forbidden` 충돌 | `forbidden` 비어 있어 직접 충돌 없음; `refactor_phase`와 테스트 약화 금지 중복 가능성만 존재 |
| `ai_behavior` | 비어 있어 검토 대상 없음 |

**완성 후:** 8섹션 모두 채움. `forbidden.implementation_without_red`와 `tdd_rules.red_phase`는 상호 보완 관계(충돌 아님).

## 4. `.cursorrules` 섹션 요약

### 4.1 TDD Rules (3 Phase)

| Phase | description | 핵심 must_not |
|-------|-------------|---------------|
| **red_phase** | 테스트 선작성, pytest RED 확인 | RED 없이 production 코드 |
| **green_phase** | 최소 구현으로 GREEN | 리팩터링, 불필요 추상화 |
| **refactor_phase** | GREEN 유지하며 구조 개선 | 테스트 약화, 동작 변경 |

### 4.2 Forbidden (6항목)

| ID | pattern | alternative |
|----|---------|-------------|
| `print_debugging` | `print()` 디버깅 | pytest assertion, logging |
| `hardcoded_magic_constant` | 34, 16 리터럴 산재 | `MagicConstant` VO |
| `bare_except` | `except:` / pass | 구체 Domain 예외 + Error schema |
| `implementation_without_red` | 실패 테스트 없이 구현 | RED → GREEN 순서 |
| `test_weakening` | assert 완화·skip·xfail | production 코드 수정 |
| `cross_layer_import` | entity → boundary/control | ECB 의존 방향 준수 |

### 4.3 Architecture — ECB 의존 방향

```
boundary → control → entity
boundary → entity (VO 변환만)

금지: entity → control, entity → boundary, control → boundary
```

## 5. 첫 Entity 구현 — User

Report/01·02에는 User 도메인 정의가 없음. **ECB Entity 레이어 학습·검증용** 최소 엔티티로 추가.

### 5.1 파일

| 경로 | 설명 |
|------|------|
| `src/entity/user.py` | User 엔티티 |
| `src/entity/exceptions/domain_errors.py` | `DomainError`, `InvalidUserError` |
| `tests/entity/test_user.py` | Logic Track pytest (10케이스) |
| `tests/conftest.py` | `src/` 경로 등록 |
| `pyproject.toml` | pytest `pythonpath`, `testpaths` |

### 5.2 User 불변조건

| ID | 규칙 |
|----|------|
| U1 | `user_id` 비어 있지 않음 (trim 후) |
| U2 | `display_name` 비어 있지 않음 (trim 후) |
| U3 | `display_name` 길이 ≤ 50 (`MAX_DISPLAY_NAME_LENGTH`) |
| U4 | 인스턴스 불변 (`frozen=True`, `slots=True`) |

### 5.3 Public API

| 메서드 | 입력 | 출력 | 실패 |
|--------|------|------|------|
| `User(user_id, display_name)` | `str`, `str` | `User` | `InvalidUserError` |
| `with_display_name(display_name)` | `str` | 새 `User` | `InvalidUserError` |

### 5.4 ECB 준수

- **Entity 레이어 only:** Boundary/Control import 없음
- type hints + Google docstring 적용
- pytest AAA 패턴, Logic Track (Domain Mock 없음)

### 5.5 구현 시 발견 사항

| 이슈 | 원인 | 조치 |
|------|------|------|
| `ModuleNotFoundError: entity.exceptions` | `tests/entity/__init__.py`가 `src/entity`와 패키지명 충돌 | `tests/entity/__init__.py` **삭제** |
| dataclass `MAX_DISPLAY_NAME_LENGTH` | 기본값 필드로 해석됨 | `ClassVar[int]` 로 변경 |
