## 2. Level 1 — Epic (Business Goal)

### 2.1 Epic Title

**불변식 기반 사고 훈련 시스템 구축** (Magic Square 4×4 TDD Practice)

### 2.2 Business Goal

4×4 Magic Square **부분 격자(빈칸 2개)** 문제를 도메인으로, **불변식(Invariant) 중심 설계·검증·구현 체계**를 갖춘 소프트웨어를 완성한다. 핵심 가치는 “정답 프로그램”이 아니라 **Concept → Invariant → Contract → Test → Component** 추적 가능한 시스템 구축이다.

### 2.3 Learning Goal

| 영역 | 목표 |
|------|------|
| 불변식 중심 설계 | I1~I11을 컴포넌트·유스케이스에 귀속 |
| Dual-Track TDD | Logic Track(Domain Mock 금지) vs UI Track(Mock 허용) |
| 입출력 계약 | E001~E007, Input/Output schema |
| TDD 사이클 | RED → GREEN(최소) → REFACTOR(계약 유지) |
| 추적성 | 주요 Invariant마다 ≥1 Test ID |

### 2.4 Success Criteria (Epic)

| ID | 기준 |
|----|------|
| SC-01~02 | 유효 입력 → int[6]; invalid → E001~E005; Domain 불가 → E006 |
| SC-03 | Domain Logic 커버리지 ≥ 95% |
| SC-05 | REFACTOR 후 외부 계약·E001~E007 message 불변 |
| SC-06~07 | 매직 넘버·정답 하드코딩 금지 |
| SC-10~12 | I1~I11 Test 추적; Traceability Matrix 유지 |

### 2.5 Key Invariants (I1~I11)

| ID | 불변식 | 검증 주체 |
|----|--------|-----------|
| I1~I5 | 완전 격자: 행·열·대각 합=M, 값={1..16}, M=34 | `MagicSquareValidator` |
| I6 | 빈칸 2개, row-major 1-index | `EmptyCellLocator` |
| I7, I11 | 누락 2개, smaller≤larger, 0 제외 | `MissingNumberFinder` |
| I8~I10 | Step A/B, Unsolvable | `TwoCellSolver` |

### 2.6 Candidate User Stories (Level 2용)

US-C01~US-C15 — Value Object, Validator, Locator, Finder, Solver, Use Case, Boundary, Dual-Track, Integration, Traceability 등 (Level 3에서 US-01~05로 수렴).

## 4. Level 3 — User Stories

### 4.1 Story Overview

| Story ID | Story Name | Layer | Protected Contract / Invariant |
|----------|------------|-------|--------------------------------|
| **US-01** | 입력 검증 | Boundary | E001~E005, U-FLOW-02 |
| **US-02** | 빈칸 좌표 탐색 | Domain | I6 |
| **US-03** | 누락 숫자 탐색 | Domain | I7, I11 |
| **US-04** | 마방진 검증 | Domain | I1~I5 |
| **US-05** | 두 가지 조합 시도 | Domain + Output | I8~I10, Output int[6] |

### 4.2 US-01 — 입력 검증 (Boundary)

**Acceptance Criteria (요약):**

- null → E003, Domain 0회
- size ≠ 4×4 → E001, Domain 0회
- count(0) ≠ 2 → E002, Domain 0회
- 값 ∉ {0}∪{1..16} → E004, Domain 0회
- non-zero duplicate → E005, Domain 0회
- valid → Domain **1회**; 검증 순서: null → size → empty → range → duplicate

**Future RED:** U-IN-01~08, U-FLOW-01~02, IT-E01~E02, IT-E05

### 4.3 US-02 — 빈칸 좌표 탐색 (Domain)

**Acceptance Criteria (요약):**

- `0`만 빈칸; 정확히 2좌표; row-major; **Domain 1-index**
- G1 → first=(2,2), second=(3,3)
- 빈칸 ≠ 2 → `InvalidGridStateError`

**Future RED:** D-LOC-01~05

### 4.4 US-03 — 누락 숫자 탐색 (Domain)

**Acceptance Criteria (요약):**

- `0` 제외 present; `{1..16}\present` = 2개; smaller ≤ larger
- G1 → 7, 10; 누락 {1,16} → 1, 16
- invalid set → `InvalidNumberSetError`

**Future RED:** D-MIS-01~03

### 4.5 US-04 — 마방진 검증 (Domain)

**Acceptance Criteria (요약):**

- `0` 포함 → false; I1~I5 모두 만족 → true; 하나라도 fail → false
- G0 → true; 행/열/대각/중복/0 각 false 케이스
- M=34는 `MagicConstant` SSOT

**Future RED:** D-VAL-01~06

### 4.6 US-05 — 두 가지 조합 시도 (Domain)

**Acceptance Criteria (요약):**

- Step A → Validator; success → (n1,n2)=(smaller,larger)
- Step A fail → Step B; success → (n1,n2)=(larger,smaller)
- G1 → [2,2,7,3,3,10]; G2 → [3,3,6,4,4,1]; G3 → Unsolvable
- length=6; 1-index; 정답 하드코딩 금지; E006 매핑

**Future RED:** D-SOL-01~03, U-OUT-01~03, IT-01, IT-03, IT-E03

## 9. G2 Fixture 공식 정의 (Report/02 부록 갱신)

Report/02 부록 G2를 아래로 **공식 확정**한다 (SC-DOM-SOL-001, D-SOL-02, IT-03).

| | c1 | c2 | c3 | c4 |
|---|-----|-----|-----|-----|
| r1 | 16 | 2 | 3 | 13 |
| r2 | 5 | 11 | 10 | 8 |
| r3 | 9 | 7 | **0** | 12 |
| r4 | 4 | 14 | 15 | **0** |

- 빈칸: `first=(3,3)`, `second=(4,4)` (1-index, row-major)
- 누락: `{1, 6}` → smaller=1, larger=6
- Step A (1→first, 6→second): **무효** (행3=29, 행4=39)
- Step B (6→first, 1→second): **유효** (I1~I5 만족)
- 기대 출력: **`[3, 3, 6, 4, 4, 1]`**
