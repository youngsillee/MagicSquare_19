본 세션은 Magic Square 4×4 프로젝트 **Boundary Track RED 착수**를 위해 AC-FR-01-01(`grid=None` / null·size 입력 검증)을 앵커로 Test Plan·RED 테스트·결함 추적 체계를 구축했다.

| Phase | 내용 | 산출 |
|-------|------|------|
| **A** | PRD FR-01 AC 샘플 예제 1건 선정 | AC-FR01-01 선택 결과 (문서) |
| **B** | pytest 기반 Test Plan | `docs/test_plan.md` |
| **C** | README RED 체크리스트 삽입 | README §RED 단계 To-Do 리스트 |
| **D** | Boundary RED 테스트 13건 | `tests/boundary/test_ac_fr_01_01_*.py` |
| **E** | venv·pytest·pytest-cov 명령 가이드 | 채팅 산출 (문서화) |
| **F** | 결함 분석·목록 | `docs/defect_list.md`, QA 분석 보고 |
| **G** | Report/Transcript Export | Report/08, Prompting/08 |

### 세션 TDD phase

| 항목 | 값 |
|------|-----|
| **현재 phase** | **RED** (실패 테스트 작성·실패 확인 완료) |
| **Track A** | Boundary — U-IN-01, U-FLOW-02 대응 (AC-FR-01-01) |
| **Track B** | 미착수 (Domain Mock 금지) |
| **GREEN** | 미진입 — `src/boundary/`, `src/control/` 미구현 |

### 계약 참고 (테스트 vs PRD SSOT)

| 항목 | 테스트 계약 (본 세션) | PRD v0.2 SSOT |
|------|----------------------|---------------|
| `grid=None` code | `INVALID_SIZE` | E003 (`INVALID_MATRIX_NULL`) |
| message | `Grid must be 4x4.` | `INVALID_MATRIX_NULL: matrix must not be null` |
| size 위반 | 동일 `INVALID_SIZE` / `Grid must be 4x4.` | E001 (`INVALID_MATRIX_SIZE: expected 4x4`) |

> 본 세션 RED 테스트는 README To-Do·사용자 지시의 **INVALID_SIZE** 계약을 따른다. GREEN 진입 시 PRD SSOT(E003/E001)와의 정합 여부를 별도 Decision으로 기록할 것.

|------|-----|
| **AC ID** | AC-FR01-01 |

| **입력** | `matrix = null` |
| **PRD 기대** | E003, Domain 0회 |

1. FR-01 검증 순서(null → size → …)상 **가장 선행**되는 입력 유효성 검사.
