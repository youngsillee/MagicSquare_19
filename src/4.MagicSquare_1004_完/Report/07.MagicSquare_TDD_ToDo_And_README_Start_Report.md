## 3. Phase A — TDD 개발 To-Do List

### 3.1 Epic

**Epic-001: 불변식 기반 사고 훈련 시스템 구축**

4×4 부분 격자(빈칸 2개) 입력 계약(E001~E007)·Domain 불변식(I1~I11)·Dual-Track TDD로 Concept→Test→Component 추적 가능한 MagicSquare_1004 완성.

### 3.2 User Stories

| Story ID | 이름 | Layer | 보호 계약 / Invariant |
|----------|------|-------|----------------------|
| **US-01** | 입력 검증 | Boundary | E001~E005, Domain 0/1회 |
| **US-02** | 빈칸 좌표 탐색 | Entity | I6, row-major 1-index |
| **US-03** | 누락 숫자 탐색 | Entity | I7, I11, 오름차순 |
| **US-04** | 마방진 검증 | Entity | I1~I5, M=34 |
| **US-05** | 두 가지 조합 시도 | Entity + Boundary | I8~I10, int[6] 출력 |

### 3.3 Tracking Board 요약 (TASK-001~038)

| Track | TASK 범위 | RED Test ID | ECB Layer | 주요 Scenario |
|-------|-----------|-------------|-----------|---------------|
| Boundary RED | TASK-001~014 | RED-BND-001~014 | Boundary | null, size, empty, range, duplicate, output, E006/E007 |
| Logic RED | TASK-015~032 | RED-DOM-001~018 | Entity, Control | D-VAL, D-LOC, D-MIS, D-SOL, UseCase orchestration |
| Integration RED | TASK-033~038 | RED-INT-001~006 | Control + Boundary | IT-01(G1), IT-03(G2), IT-E01~E05 |

### 3.4 TDD 흐름 정의 (Phase A)

```
Scenario
  → Acceptance Criteria
  → RED Test ID
  → Test Skeleton
  → Run Test (pytest)
  → Confirm Failure = RED  ← 테스트 실패 상태 확인
  → Minimal Implementation = GREEN  ← 최소 구현 작업 후보
  → Structure Improvement = REFACTOR  ← GREEN 이후 구조 개선 후보
```

| 단계 | 규칙 |
|------|------|
| **RED** | 실패하는 테스트 선작성 → `pytest`로 **실패 상태 확인** → production 코드 작성 금지 |
| **GREEN** | RED 실패를 통과시키는 **최소 구현 후보**만 수행 |
| **REFACTOR** | GREEN 유지 → 구조 개선 **후보만** 기록 → E001~E007·int[6] 계약 불변 |

## 5. Dual-Track TDD 추적 구조

### 5.1 Track 분리

| Track | 테스트 경로 | Mock | RED ID 접두 |
|-------|-------------|------|-------------|
| **Track A — Boundary RED** | `tests/boundary/` | Control/Entity Mock **허용** | `RED-BND-*` |
| **Track B — Logic RED** | `tests/entity/` | Domain Mock **금지** | `RED-DOM-*` |
| **Integration RED** | `tests/test_integration.py` | real stack | `RED-INT-*` |

### 5.2 PRD Test ID ↔ 본 세션 RED ID 매핑

| PRD SSOT (Report/02) | 본 세션 RED ID | 비고 |
|----------------------|----------------|------|
| U-IN-01~08 | RED-BND-001~008 | Boundary 입력 검증 |
| U-FLOW-01~02 | RED-BND-009~010 | Domain 호출 횟수 |
| U-OUT-01~03 | RED-BND-011~014 | 출력·에러 매핑 |
| D-VAL-01~06 | RED-DOM-001~006 | Validator |
| D-LOC-01~05 | RED-DOM-007~011 | EmptyCellLocator |
| D-MIS-01~03 | RED-DOM-012~014 | MissingNumberFinder |
| D-SOL-01~03 | RED-DOM-015~017 | TwoCellSolver |
| IT-01, IT-03, IT-E* | RED-INT-001~006 | E2E |

> **Canonical Test ID:** Report/02·PRD SSOT는 `D-*`, `U-*`. 본 세션 `RED-BND-*`, `RED-DOM-*`는 **Tracking Board alias**이며 Test Skeleton 작성 시 PRD ID와 병기 권장.
