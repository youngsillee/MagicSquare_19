# PRD — 1단계: 냉장고 이미지 입력 및 재료 인식 (Vision)

## 1. 문서 정보

| 항목 | 내용 |
|------|------|
| 제품 | 냉장고 사진 기반 재료 인식 · 레시피 추천 웹앱 |
| 단계 | 1단계 — 이미지 입력 및 재료(식재료) 인식 |
| 버전 | 0.1 (초안) |
| 의존성 | 없음 (후속 2·3단계의 입력 소스) |

---

## 2. 목표

사용자가 냉장고(또는 식료품 보관 공간) 사진을 업로드하면, **OpenRouter API**와 비전 모델 **`google/gemma-3-27b-it:free`**를 사용해 화면에 보이는 **식재료·조미료·가공식품 목록**을 구조화된 형태로 반환한다.

### 2.1 성공 기준 (1단계 완료 정의)

- 브라우저에서 이미지를 선택·업로드(또는 촬영 후 업로드)할 수 있다.
- 서버(또는 안전한 BFF)가 OpenRouter에 요청을 보내고, **API 키는 클라이언트에 노출되지 않는다**.
- 응답으로 **재료 목록**(이름 위주, 가능하면 수량/상태 힌트)을 JSON 등 머신이 읽기 쉬운 형태로 받을 수 있다.
- 이미지가 없거나, 음식과 무관한 사진이거나, 인식이 불가한 경우 **명확한 오류·안내 메시지**를 사용자에게 보여준다.

---

## 3. 사용자 스토리

1. **사용자**로서 냉장고 안 사진을 올려서, **자동으로 재료 목록**을 얻고 싶다.
2. **사용자**로서 업로드 전에 **미리보기**로 사진이 맞는지 확인하고 싶다.
3. **사용자**로서 인식이 틀렸을 때 **다시 찍거나 다른 사진**을 올릴 수 있어야 한다.

---

## 4. 범위 (In Scope)

- 이미지 파일 업로드(권장: JPEG/PNG/WebP, 최대 용량·해상도 상한 정책 정의).
- 서버 측에서 OpenRouter `POST /api/v1/chat/completions` 호출.
- 메시지 구성: `role: user`, `content`에 텍스트 지시문 + `image_url`(Base64 data URL 또는 서버가 호스팅한 임시 URL — 보안·TTL 정책 포함).
- 모델: **`google/gemma-3-27b-it:free`** 고정(1단계).
- 프롬프트: “냉장고/선반 사진에서 식용으로 보이는 항목만 나열, 불확실하면 표시” 등 **일관된 출력 스키마** 강제(예: JSON 배열만 출력하도록 요청 + 파싱 실패 시 1회 재시도 또는 완화된 파싱).
- 기본 UI: 업로드 영역, 로딩 상태, 결과 목록(태그/리스트), 오류 표시.

---

## 5. 비범위 (Out of Scope) — 1단계에서 하지 않음

- 레시피 문장 생성, 조리 단계, 영양 계산 (**2단계**).
- 사용자 계정, 로그인, DB 저장 (**3단계**).
- 여러 장 동시 분석 후 “통합 재고” (후속 확장으로 문서만 언급 가능).

## 7. 비기능 요구사항

- **보안**: API 키는 서버/BFF에만 존재. 업로드 파일은 최소 권한 저장 또는 메모리 처리 후 즉시 폐기 정책 선택.
- **개인정보**: EXIF 위치 정보 제거 권장(서버 또는 클라이언트).
- **성능**: 목표 응답 시간(예: p95 60초 이내 등)은 무료 모델 변동을 고려해 “가이드라인”으로만 명시.
- **접근성**: 키보드로 업로드 트리거 가능, 대비 충분한 오류 텍스트.

---

## 8. API 계약 (1단계 산출물 → 2단계 입력)

다음 필드를 **안정적인 계약**으로 정의해 2단계에서 그대로 사용한다.

- `ingredients[]`: `name` 필수, `note`(선택), `confidence`(선택)
- `raw_summary`(선택): 모델 자유 서술 백업
- `image_meta`(선택): 해상도, 처리 시각(디버깅용)

---

## 9. 리스크 및 완화

| 리스크 | 완화 |
|--------|------|
| 무료 모델 429/업스트림 장애 | 재시도, 사용자에게 “잠시 후 재시도” 안내 |
| JSON 이탈 | 스키마 검증, 재요청 또는 부분 파싱 |
| 오인식 | 사용자에게 “참고용” 고지, 2단계에서 수정 UI는 선택 |

---

## 10. 인수 테스트 (예시)

1. 일반적인 냉장고 사진 1장 → 3개 이상 재료가 합리적으로 추출된다.
2. 빈 접시 사진 → 재료 없음 또는 저신뢰 안내.
3. API 키 미설정 서버 → 명확한 설정 오류(내부 로그만).
